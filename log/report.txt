Отчет по домашнему заданию 

log_halley.h / log_halley.c Содержит имплементацию вспомогательной функции
log_halley() - вычисление логарифма методом Галлея, который используется при
заполнении lookup_table без использования библиотечной log(). Решает уравнение
e^y = x с помощью 5 итераций.

table.h / table.c Содержат lookup_table и их инициализацию через
init_lookup_table().
R_TABLE: содержит обратные величины средних точек i-ых интервалов мантиссы.
(Иначе: R_TABLE[i] = 1 / x_i, где x_i - середина i-го интервала). При
мантиссе близкой к 1.0 (в диапазоне 0.99 - 1.01) константа принудительно
устанавливается в 1.0 для избежания потери точности.
T_TABLE: содержит логарифм, взятый от этой точки. 
(Иначе: T_TABLE[i] = log_halley(x_i), для согласованности двух таблиц взято
-log_halley(1/x_i) по свойству логарифма). Таблицы покрывают интервал
мантиссы с шагом 1/256.

reference.sollya / logf_coeffs.h - Используются для получения констант для
полиномиальной аппроксимации. Скрипт Sollya (reference.sollya)
генерирует файл logf_coeffs.h с 5 константами полинома 5-й степени для ln(1+x).
Коэффициенты подправлены для минимизации максимальной ошибки на интервале
[-1/512, 1/512] (минимаксный метод).

log.h / log.c - Основной алгоритм

1.  Проверка аргумента на особые случаи
2.  Обработка денормализованных чисел: 
    Если число денормализованное (экспонента
    = 0, число < 0x00800000), оно умножается на 2^23
    (8388608.0f), смещение экспоненты e_offset корректируется на -23.

3.  Извлечение экспоненты и редукция мантиссы: 
    используется специально подобранный bias = 0x3f3504f3. Это
    позволяет оптимально центрировать мантиссу и минимизировать ошибки
    округления при переходе через степени двойки.

4.  Индексация таблицы: Старшие 8 бит приведенной мантиссы дают индекс: i =
    (ux_norm & 0x007FFFFF) >> 1, определяет один из 256 интервалов.

5.  Редукция аргумента: Извлекаются табличные значения rcp = R_TABLE[i] и t =
    T_TABLE[i]. Вычисляется отклонение r = m * rcp - 1.0. Вычисление
    производится с помощью аппаратной инструкции fmaf для сохранения
    точности и увеличения скорости работы.

6.  Вычисление логарифма: Используется формула ln(m) = ln(x_i) + ln(1 + r). Для
    ln(1 + r) используется полином с готовыми коэффициентами, который
    вычисляется по схеме Горнера с последовательным применением
    инструкций fmaf. Финальный результат восстанавливается по формуле:
    Result = (e + e_offset) * ln(2) + t + poly_val, также с использованием fmaf.

main.c Реализует два режима работы (через макрос TESTING):
Интерактивный режим: Ввод одного числа с клавиатуры, сравнение с GNU libm,
    вывод ULP Error и шестнадцатеричных представлений.
Режим тестирования: Проход по заданному диапазону [start_hex, end_hex], 
    подсчет метрик точности и выявление глобального максимума ошибки.
