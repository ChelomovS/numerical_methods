Аналитическая модель Блэка-Шоулса (C++)
Процесс Орнштейна-Уленбека (Python)

Реализовал расчет премии Call-опциона для 1000 различных.
Для оптимизации применил распараллеливание (С++ threads).

Результаты бенчмарка:
- Количество потоков: 8 (hardware_concurrency).
- Время выполнения (Wall time): 0.220625 ms.
- Цена первого опциона (K=80): 23.91.
- Цена последнего опциона (K=120): 0.465735.

Вывод по C++: 
Использование многопоточности позволило достичь производительности ~4.5 млн 
опционов в секунду. Линейное изменение страйка наглядно показывает переход 
цены от состояния "глубоко в деньгах" к "вне денег".

Моделирование процесса Орнштейна-Уленбека с параметрами:
T=1, v0=10, k=10, theta=50, q=1, delta=0.0001.

Анализ графика:
Визуально график напоминает логарифмическую кривую (быстрый рост в начале 
с последующим выходом на горизонтальную прямую - плато). 

Объяснение эффекта:
- Начальное значение v0=10 значительно ниже целевого уровня theta=50.
- Коэффициент скорости возврата k=10 заставляет процесс экспоненциально 
  стремиться к среднему.
- Математическое ожидание процесса описывается формулой: 
  E[vt] = theta + (v0 - theta) * exp(-kt).
- Этот экспоненциальный рост к уровню 50 придает пучку из 1000 
  траекторий форму быстрорастущей, а затем стабилизирующейся кривой.

Общие выводы
1. Многопоточность в C++ эффективна для задач с независимыми данными, к которым относится расчет цен опционов.
2. Wall time 0.22 ms подтверждает отсутствие накладных расходов на мьютексы 
   (lock-free метод).
3. Процесс Орнштейна-Уленбека визуально отображает свойство возврата к среднему
